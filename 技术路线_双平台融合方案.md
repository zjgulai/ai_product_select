# VOC AI选品平台 — 双平台融合技术路线（修正版）

> 基于用户战略确认：TikTok + Amazon 必须并行建设，最终融合为跨平台选品系统。当前分开的原因是产品映射关系尚未建立。

---

## 一、重新理解战略：为什么两个平台必须融合

### 1.1 单一平台选品的盲区

| 平台 | 能告诉你什么 | 无法告诉你什么 |
|------|-------------|---------------|
| Amazon | 什么在卖、卖多少、用户怎么评价 | 为什么突然开始卖、需求从哪里来 |
| TikTok | 什么在火、谁在讨论、情绪如何 | 火了之后有没有人愿意掏钱买 |

**盲区的代价**：只看Amazon，你在追逐已经验证的需求（红海）；只看TikTok，你在赌热度能转化（高风险）。

### 1.2 融合后的选品逻辑：TikTok发现 → Amazon验证

```
TikTok侧信号                          Amazon侧验证
─────────────────────────────────────────────────────────
视频播放量飙升  ─────────────────→  对应ASIN销量是否同步增长？
达人集中带货  ─────────────────→  商品评论数是否在快速积累？
评论区高频词  ─────────────────→  Amazon Review中是否出现相同痛点？
标签挑战赛  ─────────────────→  搜索关键词排名是否上升？
                         ↓
              融合判断：热度是"虚火"还是"真需求"
```

**核心洞察**：当TikTok热度高但Amazon供给少（评论数低、卖家数少）时，这就是最优选品窗口——需求已被社媒验证，但电商供给尚未跟上。

### 1.3 融合平台的独特价值主张

> "不是告诉你什么在Amazon上卖得好，也不是告诉你什么在TikTok上火了，而是告诉你：**什么东西正在TikTok上形成需求爆发，但在Amazon上还没被充分满足**"

这比任何单一平台工具都有壁垒。

## 二、产品映射关系：融合的技术基石

这是当前最大的技术难点，也是你为什么暂时分开做的原因。映射关系不建立，两个平台的数据就是孤岛。

### 2.1 映射关系的三种层级

```
L1: 关键词映射（最容易，先做）
    TikTok标签/文案关键词  ↔  Amazon搜索关键词
    例：#bottlewarmertiktok ↔ "baby bottle warmer"

L2: 产品概念映射（中等难度，核心）
    TikTok视频展示的产品功能/场景  ↔  Amazon类目/属性
    例："便携温奶器+户外场景" ↔  ASIN B0C1234567（momcozy）

L3: ASIN级精确映射（最难，逐步完善）
    TikTok视频中展示的具体商品  ↔  Amazon精确ASIN
    例：视频中出现的粉色温奶器（通过OCR+视觉识别）↔  ASIN
```

### 2.2 L1 关键词映射的实现方案

**数据源**：
- TikTok侧：视频文案、评论区文本、Hashtag、达人带货描述
- Amazon侧：搜索关键词、ASIN标题、Bullet Points、Review文本

**映射算法**：

```
Step 1: 分别对两边做关键词提取
    TikTok: spaCy/LLM提取名词短语 → ["bottle warmer", "portable", "travel"]
    Amazon: 同上 → ["baby bottle warmer", "portable", "fast heat"]

Step 2: 语义相似度匹配
    使用embedding模型（如BGE-multilingual）计算跨平台关键词向量相似度
    similarity("bottle warmer", "baby bottle warmer") = 0.92 → 建立映射

Step 3: 人工校验 + 反馈闭环
    初期映射由算法生成候选，运营人员确认/修正
    确认的映射存入映射库，作为训练数据优化模型
```

**数据表设计**：

```sql
-- 关键词映射表（L1）
CREATE TABLE keyword_mappings (
    id SERIAL PRIMARY KEY,
    tiktok_keyword VARCHAR(255) NOT NULL,
    amazon_keyword VARCHAR(255) NOT NULL,
    similarity_score DECIMAL(4,3) NOT NULL,  -- 语义相似度
    mapping_type VARCHAR(20),  -- 'auto' | 'manual' | 'confirmed'
    frequency INT DEFAULT 1,   -- 被多少对内容验证过
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(tiktok_keyword, amazon_keyword)
);

-- 产品概念映射表（L2）
CREATE TABLE product_concept_mappings (
    id SERIAL PRIMARY KEY,
    concept_id VARCHAR(100) NOT NULL UNIQUE,  -- 如 "portable_bottle_warmer"
    concept_name VARCHAR(255) NOT NULL,
    tiktok_keywords JSONB,      -- ["便携温奶器", "travel bottle warmer"]
    amazon_keywords JSONB,      -- ["portable baby bottle warmer", "travel bottle warmer"]
    amazon_categories JSONB,    -- ["Baby > Feeding > Bottle Warmers"]
    key_attributes JSONB,       -- {"feature": ["portable", "usb-charged"], "scene": ["travel", "car"]}
    confidence_score DECIMAL(4,3),
    created_at TIMESTAMP DEFAULT NOW()
);

-- ASIN级精确映射表（L3）
CREATE TABLE asin_tiktok_mappings (
    id SERIAL PRIMARY KEY,
    asin VARCHAR(20) NOT NULL REFERENCES products(asin),
    tiktok_video_id VARCHAR(100),
    tiktok_creator_id VARCHAR(100),
    mapping_method VARCHAR(30),  -- 'manual_link' | 'ocr_match' | 'brand_mention' | 'hashtag'
    confidence DECIMAL(4,3),
    verified BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 2.3 L2 产品概念映射的建立流程

这是融合平台的核心资产，建议按以下流程逐步积累：

**Phase 1: 种子映射（冷启动）**

1. 选择10-20个核心类目（如母婴、家居、美妆），人工建立初始概念映射
2. 每个概念包含：功能描述 + TikTok常见说法 + Amazon类目路径
3. 用这些种子数据训练/微调embedding模型

**Phase 2: 半自动扩展**

1. 系统每天扫描新增TikTok热门视频，提取产品概念候选
2. 用embedding模型在已有概念库中找最相似的，若相似度>0.8，自动归类
3. 若相似度<0.8，推送给运营人员确认是否为新概念
4. 运营确认后，系统反向搜索Amazon侧匹配的商品和关键词

**Phase 3: 全自动闭环**

1. 积累足够多映射数据后，用有监督学习训练分类模型
2. 新视频/商品自动归类到概念，准确率>90%
3. 定期扫描未映射的内容，自动发现新兴概念

### 3.2 关键融合指标定义

**Social Heat Index (SHI) — 社媒热度指数**

```
SHI = w1 × normalized(video_views) 
    + w2 × normalized(influencer_mentions) 
    + w3 × normalized(comment_engagement)
    + w4 × normalized(hashtag_growth_rate)

其中：
- video_views: 该概念下所有相关视频的播放量总和（近30天）
- influencer_mentions: 带货达人提及次数
- comment_engagement: 评论区讨论密度（评论数/播放量）
- hashtag_growth_rate: 相关Hashtag的搜索/使用增长率
```

**Commerce Validation Index (CVI) — 电商验证指数**

```
CVI = w1 × normalized(sales_volume) 
    + w2 × normalized(review_growth_rate) 
    + w3 × normalized(search_ranking)
    + w4 × normalized(seller_count)

饱和修正：
CVI_effective = CVI × (1 - saturation_penalty)
saturation_penalty = min(1, seller_count / threshold)  -- 卖家过多时降权
```

**Opportunity Score — 综合选品机会分**

```
Opportunity = SHI × (1 - CVI_effective) × trend_momentum × 100

解读：
- > 70: 极佳机会（社媒热 + 电商不饱和 + 趋势上升）
- 40-70: 值得关注（部分条件满足）
- < 40: 机会一般或风险较高

trend_momentum: 近7天SHI变化率 / 近30天SHI变化率
  > 1: 加速上升
  = 1: 匀速
  < 1: 减速
```

**Voice Gap Index — VOC差异指数（VOC核心）**

```
VGI = similarity(
    TikTok用户讨论的"期望功能"（从评论区提取），
    Amazon现有商品实际提供的"功能"（从Review中提取）
)

VGI越低 = 用户想要的 vs 市场供给的 差距越大 = 差异化机会越大
```

## 四、调整后的四阶段路线图（双平台并行）

### Phase 1: 双平台数据管道 + 种子映射（8-10周）

**目标**：两个平台都能独立采集和存储真实数据，建立第一批概念映射

**Amazon侧（与你已有页面对应）**：

| 周 | 任务 | 产出 |
|----|------|------|
| 1-2 | 接入Amazon PA API + 自研评论爬虫 | 能按ASIN/类目采集商品信息和评论 |
| 3-4 | 设计并实现Amazon侧数据模型（Product/Review/Category/Keyword） | 标准化存储，替代dynamic_data |
| 5-6 | 评论NLP基础pipeline（清洗、分词、关键词提取） | 单条评论→结构化标签 |
| 7-8 | 将现有Amazon页面接入真实数据 | 用户看到的不再是mock |

**TikTok侧（与你已有页面对应）**：

| 周 | 任务 | 产出 |
|----|------|------|
| 1-2 | 接入TikTok Research API（官方）+ 备选爬虫方案 | 能采集视频元数据、评论 |
| 3-4 | 设计并实现TikTok侧数据模型（Video/Creator/Hashtag/Comment） | 标准化存储 |
| 5-6 | 视频内容特征提取（标题/文案NLP + 评论区分析） | 视频→产品概念候选 |
| 7-8 | 将现有TikTok页面接入真实数据 | 大盘/商品/达人/视频/直播数据真实化 |

**映射层（跨平台，本阶段核心）**：

| 周 | 任务 | 产出 |
|----|------|------|
| 5-6 | 部署embedding模型（BGE），建立关键词语义匹配pipeline | L1关键词映射自动生成 |
| 7-8 | 人工标注100-200对跨平台关键词映射，验证算法准确率 | 种子映射库 + 准确率评估 |
| 9-10 | 建立产品概念表，将L1映射聚合成L2概念 | 首批20-50个核心概念 |

### Phase 2: 融合分析引擎 + MVP前端（8-10周）

**目标**：用户能输入一个概念/关键词，看到融合分析结果

**融合引擎开发**：

| 周 | 任务 | 产出 |
|----|------|------|
| 1-2 | 实现SHI计算pipeline（TikTok侧数据聚合） | 每个概念有实时热度分 |
| 3-4 | 实现CVI计算pipeline（Amazon侧数据聚合） | 每个概念有电商验证分 |
| 5-6 | 实现Opportunity Score融合计算 | 跨平台选品评分模型 |
| 7-8 | 接入LLM生成融合分析报告 | 自然语言分析报告 |

**前端融合页面（新增）**：

| 页面 | 功能 |
|------|------|
| 融合选品看板 | 展示Opportunity Score Top 100的概念，按类目筛选 |
| 概念详情页 | 单个概念的完整画像：TikTok热度趋势 + Amazon电商数据 + VOC对比 + 机会评估 |
| 融合报告生成 | 输入类目/关键词，输出AI撰写的融合选品报告 |
| 预警中心 | SHI飙升但CVI低的概念自动告警 |

**已有页面改造**：

- Amazon关键词趋势页：增加"TikTok关联热度"列
- TikTok大盘页：增加"Amazon对应品类表现"面板
- 商品详情页：如果是已映射的ASIN，展示关联的TikTok视频

### Phase 3: 产品化 + VOC深度（6-8周）

**目标**：从"数据展示"升级为"可执行的选品决策"

- **VOC深度分析**：跨平台评论对比（TikTok评论区 vs Amazon Reviews），自动识别"需求缺口"
- **竞品全景**：同一概念下，TikTok上谁在推、Amazon上谁在卖、各自表现如何
- **内容策略引擎**：基于Amazon爆款特征，生成TikTok内容脚本建议（反之亦然）
- **会员体系**：按分析概念数/报告数/监控数计费

### Phase 4: 智能化 + 生态扩展（持续）

- **预测模型**：用SHI变化预测CVI未来14天走势
- **自动映射完善**：L3 ASIN级映射通过OCR+视觉识别自动建立
- **多平台接入**：YouTube Shorts、Instagram Reels作为补充社媒信号
- **供应链对接**：高机会概念自动匹配1688/Alibaba供应商

## 八、总结

调整后的核心认知变化：

1. **不是"Amazon选品工具 + TikTok数据看板"**，而是"跨平台需求发现引擎"
2. **产品映射层是最大技术壁垒**，需要分L1/L2/L3逐步建设
3. **VOC能力是核心差异点**：不是简单的评论情感分析，而是"TikTok用户想要什么 vs Amazon卖的是什么"的跨平台差距分析
4. **两个平台不是并行独立做，而是有依赖关系**：映射层建设需要两边数据同时到位，但前期可以先独立跑通各自的数据Pipeline

最关键的第一步：**在Phase 1就同时启动两边的数据采集，并在Week 5-6开始交叉做关键词映射**。不要等两边都"完美"了再做融合——映射关系的建立本身需要两边的真实数据来驱动。
